Hexo 学习笔记

Hexo 是一款快速、简洁且高效的博客框架。它使用 Markdown（或其他渲染引擎）解析文章，在几秒内即可利用靓丽的主题生成静态网页。

准备工作

在开始使用 Hexo 之前，你需要确保你的电脑上已经安装了以下软件：

  • Node.js (建议使用 LTS 版本)
  • Git

Hexo 常用命令

以下是一些 Hexo 的常用命令：

命令 缩写 描述
hexo init [folder] hexo i 新建一个网站。如果未指定 folder ，则在当前文件夹下初始化。
hexo new [layout] <title> hexo n 新建一篇文章。如果未指定 layout，则使用 _config.yml 中的 default_layout 参数代替。
hexo generate hexo g 生成静态文件。
hexo publish [layout] <filename> hexo p 发表草稿。
hexo server hexo s 启动服务器。默认情况下，访问网址为： http://localhost:4000/。
hexo deploy hexo d 部署网站。
hexo render <file(s)> hexo r 渲染文件。
hexo migrate <type> 从其他博客系统迁移内容。
hexo clean 清除缓存文件 (db.json) 和已生成的静态文件 (public 文件夹)。
hexo list <type> hexo l 列出网站资料。
hexo version hexo v 显示 Hexo 版本信息。

hexo init

用于初始化一个新的 Hexo 项目。

这会在当前目录下创建一个名为 my-awesome-blog 的文件夹，并包含所有 Hexo 网站的初始文件。

hexo new

用于创建一篇新的文章。

这会在 source/_posts 文件夹下创建一个名为 我的第一篇博客文章.md 的 Markdown 文件。

你也可以指定文章的布局（layout），默认为 post。Hexo 内建了三种布局：post、page 和 draft。你也可以在 scaffolds 文件夹中自定义布局。

这会在 source 文件夹下创建一个名为 关于我 的文件夹，其中包含一个 index.md 文件，通常用于创建独立页面（例如“关于”页面、“友链”页面等）。

hexo generate

用于生成静态网站文件。

或者使用缩写：

生成的静态文件会存放在 public 文件夹中。这个文件夹包含了你的网站的所有 HTML、CSS、JavaScript 和图片等资源。

常用选项：

  • -d, --deploy: 生成完成后立即部署网站。
  • -w, --watch: 监视文件变动并自动重新生成。

hexo server

用于启动本地开发服务器。

或者使用缩写：

默认情况下，服务器会运行在 http://localhost:4000/。你可以在浏览器中打开这个地址来预览你的网站。

常用选项：

  • -p, --port: 指定服务器端口号。例如：hexo server -p 5000
  • -s, --static: 只提供静态文件服务。
  • -l, --log: 开启日志记录，显示更为详细的服务器信息。

hexo deploy

用于将你的网站部署到远程服务器。

或者使用缩写：

在部署之前，你需要在 _config.yml 文件中配置好部署设置（例如 Git、GitHub Pages、Heroku 等）。

常用选项：

  • -g, --generate: 部署前预先生成静态文件。

hexo clean

用于清除缓存文件和已生成的静态文件。

当你遇到一些奇怪的问题，例如修改了配置或主题但没有生效时，可以尝试运行此命令。它会删除 public 文件夹和 db.json 文件。

Hexo 写作

Hexo 使用 Markdown 格式书写文章。

Front-matter

Front-matter 是 Markdown 文件最上方以 --- 分隔的区域，用于指定个别文件的变量。例如：

常用的 Front-matter 选项：

  • title: 文章标题
  • date: 文章创建日期 (格式：YYYY-MM-DD HH:mm:ss)
  • updated: 文章更新日期 (格式：YYYY-MM-DD HH:mm:ss)
  • tags: 标签 (可以是一个或多个)
  • categories: 分类 (可以是一个或多个，支持层级)
  • permalink: 文章的永久链接
  • layout: 布局 (默认为 post)
  • draft: 是否为草稿 (设为 true 则不会被生成到 public 目录)

标签和分类

  • 标签 (Tags): 用于描述文章的关键词，没有层级关系。一篇文章可以有多个标签。
  • 分类 (Categories): 用于组织文章，具有层级关系。一篇文章可以属于多个分类。

Hexo 配置

Hexo 的主要配置文件是项目根目录下的 _config.yml 文件。你需要根据自己的需求修改这个文件中的配置项。

常用配置项：

  • title: 网站标题
  • subtitle: 网站副标题
  • description: 网站描述
  • keywords: 网站关键词
  • author: 你的名字
  • language: 网站语言 (例如：zh-CN, en)
  • timezone: 网站时区
  • url: 网站网址 (非常重要，部署时会用到)
  • root: 网站根目录
  • permalink: 文章的永久链接格式 (例如：:year/:month/:day/:title/)
  • theme: 当前使用的主题名称 (需要将主题文件放在 themes 目录下)
  • deploy: 部署相关的配置

修改配置后，可能需要重新运行 hexo clean 和 hexo generate 才能生效。
